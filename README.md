# 🏠 РемонтПро — Учебный сайт для практики Manual QA

Многостраничный сайт компании по ремонту квартир с бэкендом.

---

## 🚀 Запуск

### 1. Установить зависимости
```bash
cd backend
npm install
```

### 2. Запустить сервер
```bash
node server.js
```

### 3. Открыть в браузере
```
http://localhost:3000
```

## 📋 REST API (для Postman)

**Base URL:** `http://localhost:3000/api`

### Авторизация
| Метод | URL | Описание |
|---|---|---|
| POST | /auth/login | Вход. Body: `{email, password}` |
| POST | /auth/register | Регистрация. Body: `{name, email, password}` |
| GET | /auth/me | Текущий пользователь. Header: Bearer токен |
| POST | /auth/logout | Выход. Header: Bearer токен |

### Заявки
| Метод | URL | Доступ |
|---|---|---|
| GET | /orders | Авторизован |
| GET | /orders/:id | Авторизован |
| POST | /orders | Публично |
| PATCH | /orders/:id/status | Авторизован. Body: `{status}` |
| DELETE | /orders/:id | Только admin |

### Услуги
| Метод | URL | Доступ |
|---|---|---|
| GET | /services | Публично |
| GET | /services/:id | Публично |

### Калькулятор
| Метод | URL | Описание |
|---|---|---|
| POST | /calculate | Body: `{serviceId, area}` |

### Пользователи
| Метод | URL | Доступ |
|---|---|---|
| GET | /users | Только admin |
| DELETE | /users/:id | Только admin |

### Health check
| Метод | URL |
|---|---|
| GET | /api/health |

## 🐛 Список намеренных багов

### Визуальные баги
1. **Главная — Статистика vs Текст**: В блоке статистики написано "5 лет гарантии", а в тексте "Гарантия 3 года"
2. **Услуги — Гарантия**: На странице /services.html указано "5 лет гарантия" в одном месте, "3 года" в других
3. **Главная — Placeholder телефона**: Поле показывает `8 (999) 000 00-00`, но валидация требует `+7` формат
4. **Футер — Copyright год**: Написано © 2022 вместо актуального года
5. **Футер — Битая ссылка**: "Блог" ведёт на несуществующую `/pages/blog.html`
6. **Мобильная версия**: Кнопка "Перезвоните мне" выходит за границы экрана
7. **Форма регистрации**: Поле email имеет тип `type="text"` вместо `type="email"`
8. **Панель управления — Таблица**: Колонка "Площадь, м²" выровнена влево вместо вправо
9. **Портфолио — Карточка #4**: Только нижние углы скруглены, верхние острые
10. **Контакты — Карта**: Внутренний блок имеет width:120% и выходит за контейнер

### Функциональные баги (API/логика)
1. **Валидация телефона**: Placeholder показывает `8 (999)...` но принимается только `+7 (999)...`
2. **Минимальная длина пароля**: Интерфейс регистрации пишет "Минимум 6 символов", но бэкенд не проверяет
3. **Калькулятор**: Принимает отрицательные числа и 0 — результат: отрицательная сумма
4. **Дашборд — Выручка**: В метрике суммируются ВСЕ заявки включая отменённые

### Баги безопасности (для продвинутой практики)
5. **Нет rate limiting**: Можно брутфорсить пароли без ограничений
6. **Case-sensitive email**: `Admin@remont-pro.ru` ≠ `admin@remont-pro.ru`
7. **Смена статуса**: Любой авторизованный пользователь (включая клиента) может менять статус чужих заявок
8. **Logout**: Не инвалидирует JWT токен на сервере — токен продолжает работать
9. **Самоудаление**: Администратор может удалить сам себя через `/api/users/:id`
10. **Нет валидации email**: При регистрации принимается `notanemail` как корректный email

## 🧪 Тестовые данные

**Статусы заявок:** `pending`, `in_progress`, `completed`, `cancelled`

**Postman — пример Login:**
```json
POST http://localhost:3000/api/auth/login
Content-Type: application/json

{
  "email": "admin@remont-pro.ru",
  "password": "admin123"
}
```

**Ответ содержит JWT токен**, который нужно подставить в заголовок:
```
Authorization: Bearer <token>
```
